    想着去阅读一些优秀的代码，看很多人推荐LevelDB项目。
    作为一名新手，那就先从编译使用开始。
    参考:
    LevelDB官方源码
    LevelDB深入浅出之整体架构
    leveldb 日常使用

    1 编译官方代码

    建议参考官方的文档LevelDB

    1.1 获取官方资源

    git clone --recurse-submodules https://github.com/google/leveldb.git

    注意：

    • --recurse-submodules 需要加上这个参数，不然后可能导致编译失败(LevelDB使用googletest)

    1.2 Build for Linux

    mkdir -p build && cd build
cmake -DCMAKE_BUILD_TYPE=Release .. && cmake --build .

    2. 编写测试代码

    测试代码参考自leveldb 日常使用

    2.1 代码

    #include <cassert>
#include <iostream>
#include <string>

#include "leveldb/db.h"
#include "leveldb/options.h"

using namespace std;

int main(void) {
  leveldb::DB *db;
  leveldb::Options options;
  options.create_if_missing = true;

  // Open
  leveldb::Status status = leveldb::DB::Open(options, "./testdb", &db);
  assert(status.ok());

  string key = "keytest";
  string value = "This is the value of keytest";

  // write
  status = db->Put(leveldb::WriteOptions(), key, value);
  assert(status.ok());

  // read
  status = db->Get(leveldb::ReadOptions(), key, &value);
  assert(status.ok());

  std::cout << "value of " << key << " is " << value << std::endl;

  // delete
  status = db->Delete(leveldb::WriteOptions(), key);
  assert(status.ok());

  status = db->Get(leveldb::ReadOptions(), key, &value);
  assert(!status.ok());

  // close
  delete db;

  return 0;

}

    2.2 编译测试代码

    g++ -o test test.cpp leveldb/build/libleveldb.a -I leveldb/include/ -lpthread -lsnappy
    • leveldb/build/libleveldb.a 为第一步编译出的文件
    • leveldb/include/ leveldb头文件位置
    • -lpthread
    • -lsnappy 解决undefined reference to leveldb::Options::Options()
    • Data is automatically compressed using the Snappy compression library.

    2.3 运行测试代码

    ./test
    正常运行后，在当前目录生成testdb目录,里面包含000003.log,CURRENT,LOCK,LOG, MANIFEST-000002等文件
